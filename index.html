How Our Solution Works

We have created a software which employs pseudoanonymization (data masking) techniques to protect sensitive data stored in relational databases. There are different types of such sensitive data: names, birth dates, locations, credit card details, etc. Pseudoanonymization replaces identifiable data with placeholder values, rendering clients anonymous.

Steps to Implement:

  • Identify Sensitive Data: Mark (manually or automatically) which database columns require data masking.
  • Perform Data Masking: Apply data masking to the marked sensitive columns.
